A Translator from Modula-2 to Component Pascal / Oberon

This translator is a port of NW's translator from V2 (?) to component pascal.
I found it in the mid 1990ies at the ftp Server of the GWDG (Gesellschaft für 
wissenschaftliche Datenverarbeitung Göttingen, part of the Max-Planck Gesellschaft)
as Apple binhex files (extension hqx) in the subdirectory 
https://ftp.gwdg.de/pub/languages/oberon/ftp.inf.ethz.ch/OberonV4/MacII/Sources/ModulaToOberon/
(in April 2023 still available). NW's original did ignore/remove comments and
gave only error numbers, which had to be looked up. I modified it in so far that 
is is now able to keep comments (partially doubling them when the comments are 
both in a DEFINITION and a corresponding IMPLEMENTATION MODULEs) and providing 
verbose error messages. 

The source of the converter modules (M2S.Eth and Modula.Eth) is given as a 
Linz-V4 file, which can be imported by the standard V4 importer. Linz-V4 is 
used since it allows version elements, i.e. keeping the V4 and the CP versions 
in the same source). PrivTexts is a crude implementation of a glue module, which 
simulates some basic features of V4's Texts. It is a standard BlackBox document. 
If M2S.Eth and Modula.Eth are stored in Linz-V4 this should be done with version 
element set to CP and fully expanded.
